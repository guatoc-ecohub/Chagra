## 1. Lo que hay que entender antes de leer la tabla

### 1.1. El 3-dot engaña en este repo

`dev` se promueve a `main` por **squash-merge**. Eso hace que el ancestro común de
`main` y `dev` sea antiquísimo, y por tanto:

```
git diff --shortstat origin/main...origin/dev -- src/   →  932 archivos, +159596, -3351   ← RUIDO
git diff --shortstat origin/main   origin/dev -- src/   →   55 archivos,  +1483, -1220    ← REAL
```

`main` y `dev` son casi idénticos en `src/`. El `+159596` es contenido que `main` **ya tiene**.

**Métrica que usé para dictaminar:** el lado de **deleciones del 3-dot contra la base real**
(eso es lo único que un merge realmente quita de la base) contrastado con el 2-dot
(divergencia de árbol, que delata rama vieja).
